GitUptimeHub とは？

GitUptimeHub は、GitHub Actions を実行基盤として利用し、 サービスの稼働状況を監視・記録・公開するためのオープンソースプロジェクトです。

外部の監視SaaSや専用サーバーを一切必要とせず、 GitHub リポジトリひとつで「公式ステータスページ」を持つことができます。

なぜ GitUptimeHub なのか

Zero Server

サーバー不要

常駐プロセスやVPSは不要。 GitHub Actions の cron 実行のみで監視が完結します。

OSS

完全オープンソース

挙動はすべてコードで確認可能。 ブラックボックスな監視とは無縁です。

Cost Free

無料で運用可能

GitHub Free プランの範囲内で運用可能。 学生・個人開発に最適です。

主な機能

HTTP / HTTPS 監視

WebサイトやAPIエンドポイントの疎通を定期チェック。 ステータスコード・応答結果を記録します。

履歴データの蓄積

各チェック結果は JSON としてリポジトリに保存。 過去の稼働状況を後から確認できます。

ステータスページ自動生成

蓄積された結果から HTML を生成し、 GitHub Pages で即時公開。

外部ステータスAPI対応

Discord / GitHub / Cloudflare / OpenAI などの 公開ステータスAPIを統合表示できます。

利用シーン

個人開発・学生プロジェクト

自作サービスに「それっぽい公式ステータスページ」を。

小規模チーム

Slack通知や重厚な監視は不要。 まずは可視化から。

OSS プロジェクト

利用者に現在の稼働状況を 透明性高く共有できます。

Upptime との違い

  • HTML生成・構造を自由に改造可能
  • 独自ステータスAPI統合を前提設計
  • 日本語環境での利用を強く意識
